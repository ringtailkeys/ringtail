---
title: MCP tools
description: The exact tool surface a coding agent drives over the daemon's MCP connection. Every tool returns names + status, never a value.
---

These are the tools the daemon exposes over MCP (Streamable HTTP, `127.0.0.1/mcp`, session
token required). The whole surface is the generative-UI vocabulary: the agent supplies
structured content; the daemon owns the pixels and the creds. **No tool returns a secret
value** ‚Äî that is enforced by `check:no-leak`.

## Plan & status

- **`plan()` ‚Üí `Status[]`** ‚Äî scan the manifest + connected state ‚Üí the grid (providers √ó
  `local/dev/staging/prod`). Emits key **names** and status, never values. Also carries
  `pendingUserMessages` ‚Äî see [Chat](#chat-the-direction-channel).
- **`updateStatus(provider, env, status)`** ‚Äî flip one grid cell to a credential status.

## Author & render the UI

- **`authorWizard(provider)` ‚Üí `Wizard`** ‚Äî derive the curated recipe's on-brand 1-2-3 setup
  wizard (`open-url` ‚Üí `paste` ‚Üí `provision`) and push it to the cockpit. Returns step names +
  kinds.
- **`renderWizard(wizard)`** ‚Äî push an agent-authored `Wizard` (the long-tail fallback) to the
  cockpit. Schema-validated; malformed is rejected.
- **`mapActions(actions)` ‚Üí `Action[]`** ‚Äî map the repo-specific + cross-tool next steps into
  the actions panel. Each `Action` is schema-validated. Returns names + prerequisites + danger
  + executor.
- **`renderActions(actions)`** ‚Äî re-render the living action panel as the user steers it.

## Complete & execute

- **`submitStep(stepId, value?)`** ‚Äî the **one** inbound value path. For a `paste` step the
  value flows user ‚Üí daemon (validated + stored) and is **never echoed back**. Returns status +
  var name only. When the next step is a safe `auto` step, the daemon **auto-advances** it
  (mint ‚Üí validate-after-mint ‚Üí provision ‚Üí sync) with no agent round-trip ‚Äî the result carries
  the value-free `autoAdvanced` outcome.
- **`executeStep(stepId)`** ‚Äî the daemon runs the provisioning loop (mint ‚Üí validate-after-mint
  ‚Üí provision ‚Üí sync) with the stored creds. Returns status + key names.
- **`executeAction(id, confirmed?)`** ‚Äî run a mapped action through the gates (prereqs ‚Üí
  hard-confirm ‚Üí dispatch). A `destructive` action is refused unless `confirmed: true`. Returns
  status + key names.
- **`mintKey(action, env?)`** ‚Äî the **generic dynamic executor**. The agent authors
  an HTTP action inline ‚Äî `{ providerAccount, method, url, headers (with a `{{ROOT}}` placeholder),
  body?, extract? }` ‚Äî and the daemon resolves the root key from the global vault, **enforces the
  domain allowlist** (a non-allowlisted host is rejected before any HTTP; redirects off the
  allowlisted host are refused, not followed), runs the call, and ‚Äî if `extract` names a minted
  secret ‚Äî files it in the sink under `ringtail/<repo>/<env>/<provider>` naming. One path covers
  **mint**, read-only **permission-check** (no `extract`), and cross-provider **wire**. A
  consequential action (e.g. a `DELETE` that spends the root key destructively) comes back
  `needs-confirm` ‚Äî the agent gets **no** `confirmed` argument, so it can never self-approve; a
  human confirms out-of-band. Returns `{ providerAccount, varName?, status, reason? }` ‚Äî **never a
  value** (root or minted; any reflected secret is scrubbed from `reason`). The root key itself is
  pasted by the user via `POST /api/root`, never through the agent.

## Chat (the direction channel)

- **`sendChat(message, choices?)`** ‚Äî agent ‚Üí user in the dashboard chat, optionally with
  tappable choice pills. Text/intent only, never a value.
- **User ‚Üí agent is event-driven (no poll tool).** A message the user types (`POST /api/chat`)
  is queued and delivered as **`pendingUserMessages: string[]`** on the return of the agent's
  next `plan` / `executeStep` / `updateStatus` / `authorWizard` call ‚Äî intent text only, never a
  value. The agent re-renders actions/wizards to match on its next natural tool call.

<Callout type="warn">
Notice what's absent: there is no `getSecret`, no `readValue`, no tool that hands the agent a
credential. The agent's entire world is names, statuses, and the UI it authors. See
[The guarantee](/docs/concepts/the-guarantee).
</Callout>
